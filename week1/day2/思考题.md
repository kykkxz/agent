# 1
竞争问题，核心在读写冲突，因为cpu的乱序执行和不同线程可能同时操作这个对象，如果在同一时间两个线程对其一个读一个写则会产生脏数据，问题一般发生在写操作上，如果两个操作同时是读的话，则不需要锁

# 2
多线程是计算场景优化的几个方向，就比如图形学最基础的光栅化算法，特点就是屏幕上每个像素都独立，A像素的结果不需要等待B像素，因此每个像素可以经行并行的计算，带入到其他计算场景，A对B如果不存在依赖，可以尝试并行优化，不过后续也需要考虑数据竞争问题和内存问题

# 3
面向对象中接口和实现分离的思想，在使用时只需要拿到基类接口，其调用的哪个子类的实现可以忽略，而其中最基本的接口可以抛出NotImplementedError，防止自类忘记定义，比如日志类可以定义一个log来输出，后续定义的自类都需要实现这个接口，可以通过NotImplementedError来提醒开发者，不然通过父类调用时肯定出错
