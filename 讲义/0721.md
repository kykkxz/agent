## 模块一.ORM概念+环境搭建+数据库连接

#### 1.理论知识

##### 1.1 什么是ORM？

对象关系映射：将python对象和数据库数据表做自动映射，不需要手动写原生SQL语句，通过面向对象的方式来操作数据库

核心对应关系：

-  Python类-->数据库数据表

- 类属性-->数据字段

- 类实例对象-->数据表中的一行数据 example/instance

##### 1.2 为什么要选SQLAlchemy？

- 兼容多数据库：mysql/sqlite/PostgreSQL(有配套的向量数据库)，切换数据库不需要改业务代码

- 杜绝SQL注入，安全性是高于原生的SQL拼接

- 代码简洁，面向对象，可读性/可维护性极高

- Python后端（django（曾经的王者）/flask/fastapi（主流）/tornado），爬虫，数据分析项目主流标配ORM框架

##### 1.3 核心组件：

- create_engine:创建数据库连接引擎（连接核心入口）

- declarative_base:创建模型基类，所有的数据库表模型都需要继承他

- sessionmaker:创建数据库绘画，所有的增删改查都是通过会话完成

#### 4.1 理论知识

1）新增核心逻辑

- 创建模型对象->db.add(对象)添加到对话->db.commit（）提交事务

- 批量新增：db.add_all([对象1，对象2])

2）查询核心方法(常用)

- db.query(模型).all():查询所有数据

- db.query(模型).first()：查询第一条数据

- db.query(模型).get(主键值)：通过主键来查询单条数据

- filter(模型.字段==值):条件查询

- filter_by(字段=值)：简易条件查询（匹配）

```
#filter
res1 = db.query(Student).filter(Student.name == "李四").all()
#filter_by
res2 = db.query(Student).filter_by(name = "李四").all()
```

**注意**：

- 大于、小于、模糊、不等于、or、多条件、联表查询：用filter

- 复杂业务有限用filter查询，不会受阻

```
# 分数大于80，filter_by做不到
res = db.query(Student).filter(Student.score > 80).all()

# 模糊匹配姓名带“王”
res = db.query(Student).filter(Student.name.like("%王%")).all()

# 不等于
res = db.query(Student).filter(Student.gender != "男").all()

#或条件 OR → 只能用 filter
from sqlalchemy import or_
# 年龄18 或者 分数90以上
res = db.query(Student).filter(or_(Student.age == 18, Student.score > 90)).all()
```

### 2. SQLAlchemy异步基础

##### 2.1 同步vs异步对比

| 特性   | 同步 ORM             | 异步 ORM               |
| ---- | ------------------ | -------------------- |
| 阻塞   | 数据库 I/O 阻塞线程       | 等待时释放 CPU            |
| 适用场景 | 单任务、低并发            | 爬虫、批量入库、多任务          |
| 写法   | `Session()`        | `AsyncSession()`     |
| 执行   | `db.execute()`     | `await db.execute()` |
| 连接串  | `mysql+pymysql://` | `mysql+aiomysql://`  |

### 2.2 异步引擎创建

```python

# 3.异步建表

# 4.异步插入数据
```

```python
def generate_jobs(count=1000):

    ]
    jobs = []
    for i in range(count):
```
